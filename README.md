# 有限要素解析結果可視化ツール

このPythonスクリプト群は、特定のフォーマットで出力された有限要素解析（FEM）の結果テキストファイルを読み込み、Matplotlibを使用してグラフとして可視化するためのツールです。

## 概要

有限要素解析プログラムが出力する複数のテキストファイル（変位、エネルギー、収束ノルム、応力・ひずみ、温度など）を対象とし、それぞれのデータに応じたグラフを対話的に表示することができます。

## プロジェクト構成

```
.
├── main_visualizer.py       # メインの可視化実行ファイル (ユーザーインターフェース)
├── data_loader.py           # データ読み込み共通モジュール
├── plot_dis.py              # 変位/荷重データ (DIS_*.txt) の可視化用モジュール
├── plot_ene.py              # エネルギーデータ (ENE_*.txt) の可視化用モジュール
├── plot_nor.py              # 収束履歴データ (NOR_*.txt) の可視化用モジュール
├── plot_sts.py              # 応力/ひずみデータ (STS_*.txt) の可視化用モジュール
├── plot_tmp.py              # 温度データ (TMP_*.txt) の可視化用モジュール
├── DIS_****.txt             # 解析結果データファイル (例: DIS_cube250514.txt)
├── ENE_****.txt             # 解析結果データファイル
├── NOR_****.txt             # 解析結果データファイル
├── STS_****.txt             # 解析結果データファイル
├── TMP_****.txt             # 解析結果データファイル
└── README.md                # このファイル
```

## 主な機能

* **対話的なインターフェース**: `main_visualizer.py` を実行すると、どの解析ケースのどの種類のデータを表示するかを選択できます。
* **モジュール化された設計**: データ読み込み、各データタイプのプロット処理がそれぞれ独立したPythonファイルに分割されており、管理や拡張が容易です。
* **多様なプロット**:
    * 荷重-変位曲線
    * 各種エネルギーのステップ変化
    * 計算ステップごとの収束履歴 (残差ノルムなど)
    * 応力-ひずみ曲線
    * 温度のステップ変化
* **Pandasによるデータ処理**: テキストファイルの読み込みとデータ操作にはPandasライブラリを使用しています。
* **Matplotlibによるグラフ描画**: グラフの描画にはMatplotlibライブラリを使用しています。

## 前提条件

* Python 3.x
* Pandas ライブラリ
* Matplotlib ライブラリ

これらのライブラリは、pipコマンドでインストールできます。
```bash
pip install pandas matplotlib
```

## 使用方法

1.  **ファイルの配置**:
    * 上記のPythonスクリプト群 (`main_visualizer.py`, `data_loader.py`, `plot_*.py`) を同じディレクトリに配置します。
    * 可視化したい解析結果のテキストファイル群 (`DIS_*.txt`, `ENE_*.txt` など) を、`main_visualizer.py` と同じディレクトリに置くか、`main_visualizer.py` 内の `base_path` 変数をデータファイルが存在するディレクトリパスに設定します。

2.  **実行**:
    ターミナルまたはコマンドプロンプトを開き、`main_visualizer.py` があるディレクトリに移動して、以下のコマンドでスクリプトを実行します。
    ```bash
    python main_visualizer.py
    ```

3.  **対話的選択**:
    * 最初に、解析ケースのプレフィックス（例: `cube250514`）を入力するよう求められます。これにより、対応するデータファイル群が読み込まれます。
    * 次に、どの種類のデータを可視化するかをメニューから選択します。
    * プロットタイプによっては、グラフ化するデータ列のインデックス（`Step` 列を除いた0始まりのインデックス）を指定するよう求められます。利用可能な列名とインデックスがターミナルに表示されるので、それに従って入力してください。

## 各モジュールの説明

* **`main_visualizer.py`**:
    ユーザーとの対話処理、データ読み込みモジュールおよび各プロットモジュールの呼び出しを行います。プログラムのエントリーポイントです。

* **`data_loader.py`**:
    `parse_header_and_load(filepath)` 関数を定義しています。この関数は、指定されたテキストファイルを読み込み、コメント行やヘッダー情報を解析してPandas DataFrameオブジェクトとしてデータを返します。テキストファイルのヘッダー構造（特に列名の抽出）については、このモジュール内のロジックが鍵となります。

* **`plot_dis.py`**:
    変位および荷重データ (`DIS_*.txt`) の可視化関数を含みます。
    * `plot_force_displacement()`: 荷重-変位曲線を描画します。
    * `plot_displacement_vs_step()`: 指定された変位成分のステップに対する変化を描画します。

* **`plot_ene.py`**:
    エネルギーデータ (`ENE_*.txt`) の可視化関数を含みます。
    * `plot_energy_vs_step()`: 各種エネルギー（総エネルギー、弾性エネルギーなど）のステップに対する変化を描画します。

* **`plot_nor.py`**:
    収束履歴データ (`NOR_*.txt`) の可視化関数を含みます。
    * `plot_convergence_history()`: 指定された計算ステップにおける反復回数に対するノルム値の変化（収束の様子）を描画します。

* **`plot_sts.py`**:
    応力およびひずみデータ (`STS_*.txt`) の可視化関数を含みます。
    * `plot_stress_strain_curve()`: 応力-ひずみ曲線を描画します。
    * `plot_stress_vs_step()`: 指定された応力成分のステップに対する変化を描画します。

* **`plot_tmp.py`**:
    温度データ (`TMP_*.txt`) の可視化関数を含みます。
    * `plot_temperature_vs_step()`: 温度のステップに対する変化を描画します。

## 注意点と今後の拡張

* **データファイルのフォーマット**: このツールは、特定のフォーマット（コメント行が `%` で始まり、データが空白区切りであるなど）のテキストファイルを想定しています。`data_loader.py` のヘッダー解析ロジックは、ファイルの正確な構造に合わせて調整が必要になる場合があります。
* **列の選択**: プロットするデータ列は、ユーザーが対話的にインデックスで指定する必要があります。ファイルによっては多くのデータ列が含まれるため、どの列が何を表しているかを事前に理解しておく必要があります。
* **3D可視化**: このツールは主に2Dグラフによる可視化を行います。変形形状や応力コンター図などの3D可視化には対応していません。そのような可視化が必要な場合は、ParaView、VisIt、Mayaviといった専用の3D可視化ソフトウェアや、PythonのVTK、PyVistaライブラリの利用を検討してください。
* **エラーハンドリング**: ファイルが存在しない場合や、予期せぬデータフォーマットの場合などに対する基本的なエラーハンドリングは含まれていますが、より堅牢なエラー処理を追加することも可能です。
* **設定ファイル**: 将来的に、プロットのスタイルやデフォルトでプロットする列などを設定ファイルから読み込むように拡張することも考えられます。

## トラブルシューティング

* **ファイルが読み込めない**:
    * ファイルパスが正しいか確認してください。
    * `data_loader.py` の `parse_header_and_load` 関数が、対象のファイルのヘッダー構造を正しく解釈できているか確認してください。必要であれば、この関数のロジックを修正してください。
* **グラフが期待通りに表示されない**:
    * `main_visualizer.py` でプロット関数を呼び出す際に指定している列のインデックスが正しいか確認してください。
    * 各 `plot_*.py` モジュール内のプロットロジックが、対象のデータと目的に合っているか確認してください。
